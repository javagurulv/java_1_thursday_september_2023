Далее опишем один из возможных вариантов
решения задачи из предыдущего задания.

Основная идея: сделать так, чтобы сами правила
знали свои названия (FraudRule1, FraudRule2, ...)!

Тогда у правила, которое обнаружило мошенническую
транзакцию можно будет узнать его название.

Тогда можно будет обрабатывать все правила одинаково,
например объединив их в список. Проверка транзакции
будет выполняться проходом по списку правил, применением
правила к транзакции, и если будет обнаружено мошенничество,
то у самого правила можно будет узнать его название и
сформировать результат (создать объект класса FraudDetectionResult).


Добавьте в интерфейс FraudRule метод getRuleName():

    String getRuleName();

и реализуйте описанные выше изменения.
