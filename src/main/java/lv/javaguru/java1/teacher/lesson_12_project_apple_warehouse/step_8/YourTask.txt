Как смоделировать любой критерий поиска?

Критерий поиска должен отвечать на вопрос:
соответствует ли объект условиям отбора или нет!

То есть ответа всего ДВА: ДА или НЕТ. Третьего не дано!

Какой тип данных подходит для этого?

Конечно же boolean: true или false.

То есть критерий поиска должен возвращать boolean,
значение, которое отвечает на вопрос: подходит или нет
объект заданным критериям поиска.

А что критерий поиска должен оценивать?
Что это за объект такой? О каком вообще объекте идёт речь?

В нашей задаче речь идёт о .... яблоках!

На складе у фермера хранятся яблоки, наша программа
оперирует именно яблоками.

В контексте другой задачи, объектами могут быть всё что угодно!

Окей, теперь нам известно, что объект, который будет оцениваться
критерием поиска в нашем случае это яблоко (экземпляр класса Apple).

На вход критерию поиска подаём яблоко, в качестве результата
получаем boolean (true - подходит под критерий поиска,
false - не подходит).

И еще одно требование к критерию поиска - у него должно быть много
реализаций! То есть фермер хочет иметь возможности:
- искать яблоки только по цвету;
- искать яблоки только по весу;
- искать яблоки и по цвету и по весу вместе
- и так далее.

Беря во внимание все выше сказанное, как в программе смоделировать
критерий поиска?

У какой конструкции в языке программирования на Java можно создавать
много реализаций?

Да это же ИНТЕРФЕЙС!!!

У интерфейса может быть много реализаций.

Давайте определим интерфейс для моделирования критериев поиска:

interface AppleSearchCriteria {
    boolean test(Apple apple);
}

Ваше задание: создайте реализации интерфейса
AppleSearchCriteria для выбора:
- зелёных яблок;
- красных яблок;
- тяжелых яблок (> 150 г.);
- лёгких яблок (< 150 г.).
